ProjectVault API Structure and File Map

This document outlines the organization of the backend API and identifies which files contain specific API logic.

Base URL: http://localhost:8000/api
Backend Source Directory: backend/app

1. Main Entry Point
   File: backend/app/main.py
   - Initializes the FastAPI application.
   - Configures CORS middleware.
   - Includes all the routers defined below.
   - Endpoints:
     - GET / (Root check)
     - GET /api/health (Health check)

2. Authentication Endpoints
   File: backend/app/routes/auth_routes.py
   - Handles user registration, login, and profile management.
   - Endpoints:
     - POST /api/auth/register (Create new account)
     - POST /api/auth/login (Get JWT access token)
     - GET /api/auth/me (Get current user profile)
     - PUT /api/auth/me (Update profile)
     - PUT /api/auth/password (Change password)

3. Project Management Endpoints
   File: backend/app/routes/project_routes.py
   - Manages project creation, retrieval, and deletion.
   - Endpoints:
     - GET /api/projects (List all user projects)
     - POST /api/projects (Create new project)
     - GET /api/projects/{id} (Get specific project details)
     - PUT /api/projects/{id} (Update project details)
     - DELETE /api/projects/{id} (Delete project)

4. File Management Endpoints
   File: backend/app/routes/file_routes.py
   - Handles file uploads, downloads, and deletion.
   - Endpoints:
     - POST /api/projects/{id}/upload (Upload files - Creates new version)
     - GET /api/projects/{id}/files (List files in current version)
     - GET /api/projects/{id}/download (Download all files as ZIP)
     - GET /api/projects/{id}/files/{file_id}/download (Download single file)
     - DELETE /api/projects/{id}/files/{file_id} (Delete specific file)

5. Version Control Endpoints
   File: backend/app/routes/version_routes.py
   - Manages version history and restoration logic.
   - Endpoints:
     - GET /api/projects/{id}/versions (Get timeline of versions)
     - POST /api/projects/{id}/versions/{ver_id}/restore (Restore a previous version)

6. Activity/Stats Endpoints
   File: backend/app/routes/activity_routes.py
   - Handles user activity logging and dashboard statistics.
   - Endpoints:
     - GET /api/activity (Get recent activity feed)
     - GET /api/activity/stats (Get user dashboard stats: storage, project count, etc.)
     - DELETE /api/activity (Clear activity history)

7. Data Models & Config
   - Database Models: backend/app/models.py (Defines User, Project, File, Version, Activity tables)
   - Pydantic Schemas: backend/app/schemas.py (Defines request/response data structures)
   - Database Connection: backend/app/database.py
   - App Configuration: backend/app/config.py (Environment variables, constants)
